# LLMWiki 셋업 가이드

LLMWiki는 **회원님의 GitHub 비공개 저장소**를 위키로 씁니다 — 서버가 없어서 내 기록은 100% 내 GitHub 안에 있습니다. 처음 한 번만 GitHub에 연결하면 됩니다.

> 💡 **어렵지 않습니다.** 예전엔 토큰을 직접 발급해야 했지만, 이제 **"GitHub로 로그인" 한 번**이면 됩니다. 아래 1번만 따라오세요.

---

## 0. 준비물
- **GitHub 계정** (없으면 https://github.com/signup — 무료)
- **앱 설치**: 📱 폰은 https://mynameisjinhohong.github.io/llmwiki-app/ · 🖥 PC는 [INSTALL-desktop.md](INSTALL-desktop.md)

---

## 1. GitHub 연결 — "GitHub로 로그인" (권장, 제일 쉬움)

1. 앱의 **위키 탭**(또는 첫 실행 화면)에서 **"GitHub로 로그인"** 을 누릅니다.
2. 화면에 **코드**(예: `AB12-CD34`)가 뜨고 브라우저가 열립니다 → GitHub에 그 코드를 입력하고 **Authorize**(승인).
3. 승인하면 앱이 자동으로 연결됩니다. **토큰 발급·복사 불필요.** 끝!

> 데스크톱·모바일은 각각 로그인합니다(저장소가 기기별로 분리). 폰↔PC는 데스크톱의 **설정 → 모바일 연동** QR로 한 번에 동기화할 수도 있습니다.

---

## 2. 위키 만들기 / 고르기

로그인하면 어떤 위키를 쓸지 정합니다. 위키 탭에서:

- **🆕 새 위키 만들기** — "새 위키 만들기"에 이름 입력 → **앱이 알아서 GitHub 비공개 repo를 생성·초기화**합니다.
  > ⚠️ **직접 github.com에서 repo를 만들지 마세요.** 앱은 자기가 만든(위키 표식이 있는) repo만 위키로 인식합니다. 반드시 앱의 "새 위키 만들기"로 만드세요.
- **📂 내 repo에서 추가** — "내 GitHub repo에서 위키 추가" → **위키 목록**(표식 있는 것만 보임)에서 선택.
- **👥 그룹 위키 참가** — 아래 3번 참고.

여러 위키(개인·독서모임·스터디…)를 추가해두고 **위키 탭에서 탭 한 번으로 전환**할 수 있습니다.

---

## 3. 그룹으로 함께 쓰기

한 위키를 여러 명이 공유합니다(무서버 — 그냥 같은 비공개 repo에 GitHub 협업자로 붙습니다).

**관리자(위키 소유자):**
1. 위키 탭 → **👥 멤버** → 상대의 **GitHub 아이디**로 초대.

**초대받은 사람:**
1. GitHub 알림/이메일(또는 `https://github.com/소유자/저장소/invitations`)에서 **Accept invitation**(수락).
2. 앱에서 **"GitHub로 로그인"**(아직 안 했다면) → **"내 repo에서 위키 추가"** 에 그 공유 위키가 뜹니다 → 추가.
   > 안 보이면 → 5번 문제해결의 "그룹 위키가 안 보임" 참고.

각자 자기 계정·자기 기기로 붙고, **결과는 하나의 통합 위키**로 모입니다.

---

## 4. 쓰기: 캡처 → 정리(ingest)

1. **캡처** — 캡처 탭에 메모·링크·사진을 저장하면 `inbox`에 쌓입니다(아직 날것).
2. **정리(ingest)** — inbox를 잘 정리된 위키 페이지로 합성합니다. 두 가지 방법:
   - **🟢 "API 정리" (쉬움, CLI 불필요)** — 설정 → 검색 LLM에 **무료 Gemini 키**를 넣고, 정리 화면에서 **"API 정리"**. 키 발급: https://aistudio.google.com/apikey (카드 불필요).
   - **⚙️ CLI 정리 (고품질)** — 데스크톱에 **Claude Code** 설치 + `claude` 로그인 후, 정리 화면의 **"정리(claude)"**. PDF 등도 잘 처리하지만 설치·로그인이 필요합니다.
   - (그룹) 각자 **자기가 캡처한 것만** 정리됩니다 → 통합 위키에 양쪽 반영.

---

## 5. 안 될 때 (문제해결)

- **로그인했는데 내 repo(특히 새로 만든 그룹 위키)가 목록에 안 보임**
  → 예전에 넣어둔 **권한 좁은 토큰**이 남아있을 수 있습니다. 위키 탭 → **"GitHub로 로그인"으로 재연동**하면 전체 repo가 보입니다.
- **"이 레포는 위키가 아닙니다" / 수동 추가가 막힘**
  → github.com에서 직접 만든 일반 repo입니다. **"새 위키 만들기"** 로 만들거나, 위키 표식이 있는 repo를 고르세요.
- **정리 시 `spawn claude ENOENT`** (데스크톱)
  → 이 PC에 Claude Code가 없습니다. **"API 정리"**(무료 LLM 키)를 쓰거나, Claude Code를 설치하세요.
- **그룹 위키가 picker에 안 보임**
  → ① GitHub에서 **초대를 수락**했는지 ② 앱에 로그인한 계정이 **초대받은 바로 그 계정**인지 확인. 그래도 없으면 위키 탭 **고급: 수동 연결**에서 owner=관리자아이디 / repo=위키이름으로 직접 추가(로그인돼 있으면 토큰칸은 비워도 됨).
- **"Invalid token (401)" / "Repo not found (404)"** (수동 연결 시)
  → 토큰 만료·오타, 또는 소유자/저장소 철자. 재연동하거나 철자를 `github.com/소유자/저장소`와 대조.
- **"OK … (PUBLIC!)"** → 연결은 됐지만 저장소가 공개 상태. GitHub repo *Settings → General → Change visibility*에서 **Private** 권장.

## 고급: 토큰(PAT)으로 수동 연결

OAuth를 못 쓰는 경우(순수 웹 브라우저, 회사 정책 등)만 필요합니다. 위키 탭 → **"고급: 수동 연결"**:

1. **위키 repo 준비**: 반드시 **앱의 "새 위키 만들기"로 먼저 생성**(표식 필요). 공유 위키면 초대 수락.
2. **Fine-grained 토큰** 발급: https://github.com/settings/personal-access-tokens/new → 이름 입력 → Expiration 선택 → **Repository access = Only select repositories**로 그 위키 선택 → **Permissions → Contents = Read and write** → **Generate token** → `github_pat_…` 복사.
3. 앱의 고급 연결에 **owner**(소유자 아이디) / **repo**(저장소 이름) / **branch**(`main`) / **token**을 넣고 저장.

## 보안
- GitHub 토큰·LLM 키는 **이 기기에만** 저장됩니다. 외부로 안 나가고 `api.github.com` 인증에만 쓰입니다.
- 토큰·연동 QR은 **비밀번호처럼** — 공유·스크린샷 금지. 노출 시 https://github.com/settings/tokens 에서 폐기.
